## Content Usage Guidelines

### When Editing Worklogs:
1. Check this reference for actual workdays in that week
2. Check for any events scheduled in that week
3. Check for personal events (injury, doctor visits)
4. Check if week falls in team project period (5/11 onwards)
5. Fill remaining days with appropriate learning activities

### Priority Order:
1. AWS Events (full coverage)
2. Personal Events (brief professional mention)
3. Team Project (from 5/11 onwards)
4. Workdays (heavier content)
5. Home days (lighter content)
6. Fill gaps (typical AWS learning or revision)
